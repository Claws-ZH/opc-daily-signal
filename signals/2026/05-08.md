## 💰 单价 — 利基中的利基：用「门槛即定价」的奢侈品逻辑做小众 SaaS

→ **事实**：Dave Snider（前 Read the Docs 联创）在 5/7 复盘自己运营两年的 Table Slayer / Counter Slayer——为「在桌子里嵌电视玩 D&D」的一小撮玩家做的工具。他明确说：受众小到几百人，但他对开源版本之外的托管版本收费，**理由不是为了 ARR，而是为了让付费客户成为产品反馈的过滤器**——「过去一年大部分功能都是为他们加的，不是为我加的」。代码廉价、托管可线性扩展、开源永不死，所以「关店」的风险趋零；亏的只是机会成本（每周 12 小时），但工具反过来打磨了他的手艺。

🔑 **动作**：明天把当前产品的目标受众**再砍一层**——不是「indie devs」，而是「在 RPi 上自托管 Obsidian 的中文用户」这种「需要自己定制硬件/家具/工作流」的群体。在 landing page 加一道「资格门槛」（自检清单：你必须已经拥有 X / 在做 Y），然后定价直接 $19→$49/月。门槛越高的客户，反馈密度越大、流失率越低。

🔮 **二阶**：6-12 个月内，「门槛即定价」会成为 solo SaaS 的标配——通用工具（Notion/Linear 仿品）会被 LLM 平权式碾死，但「需要先买定制 PCB / 3D 打印 / 改造家具」这类前置投入的工具反而构成天然护城河，因为客户已经沉没成本，迁移成本极高。同期会出现一批「为爱好者群体做的、看起来不可能盈利但实际单价 $30-100/月」的产品。

⚠️ **反向检验**：如果三个月内你 landing page 转化率 < 0.5% 且付费用户 < 30，说明你选的不是「利基」而是「不存在」——立即换赛道。利基的下限是「至少有一个 Discord/论坛存在 1 年以上、>500 活跃成员」。

## ⚡ 时间 — Agent 产品别再堆 prompt：把控制流写成代码，调试时间砍 10x

→ **事实**：HN 302 分热帖（5/7，bsuh）核心论点：当前 agent 工程师的主要时间都耗在「微调 prompt 让模型走对流程」上，但这本质是把**确定性的控制流**塞进**非确定性的自然语言**——错误调试要看 trace、看 token、看温度，没有断点。正确做法是把 if/else / 循环 / 重试 / 工具选择写成显式 Python/TS 代码，只在「需要语义判断」的叶子节点调 LLM。GitHub Trending 同期 14k stars 的 TauricResearch/TradingAgents 和 OpenAI 自家的 symphony 都是这个范式：多 agent + 显式编排 > 单一 super-prompt。

🔑 **动作**：审一遍你正在维护的 agent 产品——把 system prompt 里所有「if X then Y」「先做 A 再做 B」的指令**翻译成 Python 函数**，LLM 只负责「给定上下文，返回一个枚举值/JSON」这一种调用。预计能砍掉 60% 的 prompt 长度、3-5x 的 token 成本，调试从「读 trace」变成「打断点」。

🔮 **二阶**：3-6 个月内，「prompt-first agent」框架（LangChain 风格）会被「control-flow-first」框架（DSPy / symphony / 自写编排）替代；招聘市场会从「prompt engineer」转向「会写状态机的工程师」。同期 AlphaEvolve 这类「LLM 优化代码」的工具会反哺 agent 控制流的自动生成。

⚠️ **反向检验**：如果你的 agent 任务**真的**只能用自然语言描述（如开放式咨询、创意生成），且测试集 pass@1 已 >85%，那 prompt-first 仍然合理——别为了显式而显式。判断标准：能否画出有限状态图。画不出，就保持 prompt。

## 📈 数量 — Skills 作为分销渠道：把方法论打包成 .md，蹭 Claude/Codex 流量

→ **事实**：mattpocock/skills 一周 16,579 stars（GitHub Trending 周榜第一），ComposioHQ/awesome-codex-skills 同周 2,287 stars。Matt Pocock 把自己 .claude 目录里的工程实践直接公开，每个 skill 是一个 SKILL.md（前置元数据 + 步骤说明），用户复制即用。本质：**他没建社区、没卖课、没做 SaaS**——只是把「我怎么干活」格式化成可被 agent 加载的标准件，然后通过 Claude/Codex 生态自动分发。stars 增速暗示这条管道当前严重供不应求。

🔑 **动作**：今天就把你最近三个月解决过的 1-2 个非平凡工程问题（部署/调试/CI 修复/特定 API 集成）写成 SKILL.md，发到 GitHub，加 `claude-skill` / `codex-skill` topic。不写代码，只写流程 + 命令 + 陷阱。一周观察 stars / 引用数——这是当前最便宜的「专业身份信号」获取方式。

🔮 **二阶**：12 个月内会出现「Skill Marketplace」——付费 skill（$5-50 一份）针对垂直领域（Stripe 集成、Postgres 调优、K8s 排错），由 agent runtime 直接拉取。早期占位的作者会形成类似「npm 包大佬」的网络效应。同期开源 skill 会变成 SaaS 的获客漏斗（免费 skill 引流到付费工具）。

⚠️ **反向检验**：如果三个月后 skill repo < 50 stars 且无任何 fork/issue，说明你的领域**已被 LLM 内置知识覆盖**（如 React 基础、SQL 语法），需换更细分的方向（如「在 Cloudflare Workers 上跑 Drizzle 的迁移陷阱」）。

## 💀 成本 — OpenClaw 翻车复盘：solo 项目走向 infra 时，「founder-driven」是慢性自杀

→ **事实**：5/5，Peter Steinberger（前 PSPDFKit 创始人）公开承认 OpenClaw 4.24-4.29 版本崩坏：plugin 依赖修复死循环、gateway 冷启动卡顿、bundled vs external plugin 半拆分状态。根本原因他自己点破：**「OpenClaw was still too founder-driven. Too much release, review, packaging and support work sat with me.」** 触发器是 npm 供应链事件（postinstall、传递依赖）让他急于「core 瘦身」，但中间态比之前更糟。解法：成立 OpenClaw Foundation + OpenAI 协助组真团队 + 推 LTS 分支。

🔑 **动作**：明天审一遍你 solo 项目的「单点故障清单」——release / review / 客服 / 域名续费 / 支付对账 / 安全响应，**每一项写下「我倒下 48 小时会发生什么」**。任何答案是「服务停摆」的，立刻找 1 个外包/合伙人接管，或者**主动缩减功能边界**（把那个模块标记为 deprecated / 移到 community plugin）。宁可砍功能，不要扛单点。

🔮 **二阶**：6-18 个月，「solo infra 工具」赛道会出现一波集中暴雷——LLM 让 1 人产出量翻倍，但运维/安全/供应链复杂度也同步翻倍，差值是负的。能活下来的会分两类：(a) 主动锁死功能边界、永不扩张的「永恒 v1」工具；(b) 早期就接受基金会/小团队治理的项目。中间态（功能在长、人还是 1 个）会全死。

⚠️ **反向检验**：如果你的项目**没有**用户群、没有依赖管理、没有 release cadence（即纯个人脚本/玩具），上面警告不适用——继续 solo 是最优解。临界点是「有第一个付费客户开始抱怨」或「依赖图 transitive 包数 > 200」，触发即必须组队或瘦身。

## 深度补充

### 底层逻辑
今日 4 条信号串起一条主线：**LLM 时代的 OPC 不再缺「能力」，缺的是「边界」**。能力（写代码、调 API、做 agent）已被平权；活下来的解法都是主动收窄——
- 💰 受众边界（小到几百人但单价拉高）
- ⚡ 工程边界（自然语言只用在叶子节点，控制流写成代码）
- 📈 输出边界（不做产品，只做 SKILL.md 这一种标准件）
- 💀 功能边界（main + LTS，砍掉一切非核心）

### 数据交叉验证
- mattpocock/skills（16k）+ ComposioHQ/awesome-codex-skills（2.3k）+ openai/symphony（2.4k）— 三个独立项目同周高位，确认「skill / 控制流」是真趋势，非单点。
- TauricResearch/TradingAgents 14k 与 virattt/dexter 3k 同为「垂直 agent + 显式编排」，与 ⚡ 信号互证。
- OpenClaw 翻车 + Xeiaso「Maybe you shouldn't install new software」(HN #48056227) 同周热议，确认 npm 供应链焦虑已蔓延到 indie 圈。

### 连锁效应
若 ⚡ 控制流范式胜出 → prompt 调试岗位贬值 → SKILL.md 取代「prompt 模板分享」成为知识流通载体（📈 验证）→ 反过来要求 solo 工具收窄边界（💀 验证）→ 利基定价成为唯一可持续模型（💰 验证）。四条信号互锁。

### 反向情景
如果 GPT-6 / Gemini 3 在 Q3-Q4 实现「prompt 即程序」（即模型自己生成稳定控制流），则 ⚡ 信号反转——但 💰📈💀 仍成立，因为它们作用于受众/分销/边界，与模型能力无关。
