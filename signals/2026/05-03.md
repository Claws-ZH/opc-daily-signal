## 💀 成本 — VS Code 在 commit 里强行注入 "Co-Authored-by: Copilot"

→ **事实**：微软 VS Code PR #310226 引发开发者炸锅——只要装了 Copilot 扩展，commit 信息会被自动加上 `Co-Authored-by: Copilot <…>`，无论这次 commit 是否真用了 AI。280+ HN 评论，主线指控：工具方未经同意把品牌烙在用户产出物的 git 历史里，永久留痕。

🔑 **动作**：今天就检查你的 git log（`git log --all --grep="Co-Authored"`）和 README/导出文件的 metadata，删除所有未经授权的工具水印。在 `~/.gitconfig` 加 `[commit] template = ~/.gitmessage`，明确只挂自己的 author。所有 AI 工具（Copilot、Cursor、Claude Code）安装后第一件事：关闭"自动 attribution"选项。

🔮 **二阶**：3-12 个月内会出现"工具方品牌寄生"的系统性事件——浏览器扩展往导出文档里塞水印、SaaS 在你的客户邮件页脚加 "Sent via XXX"、AI IDE 在生成代码注释里留 "Generated by"。OPC 个人品牌会被层层稀释。法律层面这接近"未披露的赞助内容"。

⚠️ **反向检验**：如果你做的是公开开源项目且本就欢迎工具协作（例如 docs 类）保留 attribution 反而提升透明度——别盲目删。判断标准：客户付钱给的是"你的产出"还是"你协调多方工具的产出"。

🔗 https://github.com/microsoft/vscode/pull/310226

## 深度小节

### 底层逻辑
四条信号共享一条主线：**资产化 vs 被资产化**。mattpocock 把 dotfile 资产化了；Uber 把司机日常行为资产化了；微软把你的 commit history 资产化了；Roblox 的合规事件揭示"在别人平台上你才是被资产化的那个"。OPC 的核心战略选择就是这一道线——尽量站在资产化方，远离被资产化方。

### 为什么 OPC 要关注
一人公司没有团队、没有渠道议价权，唯一的杠杆是**杠铃配置**：低成本资产化自有产出（💰⚡）+ 主动隔离外部资产化风险（📈💀）。今天的四条信号正好覆盖这两端。

### 逆向检验
最大风险是把 mattpocock 的 35k stars 误读为"个人 dotfile 必能商业化"——实际上他有十多年 TS 教育者品牌底盘，新人裸发 repo 大概率 100 stars 封顶。可复制的不是结果数字，是**"把私人工具包装成可分发资产"的动作**本身。

### 数据交叉验证
- mattpocock 35,324 vs forrestchang 20,079 stars/week — 同周双爆款，趋势确认非偶然
- Roblox -18% 与近 6 月 Online Safety Act 执法时间线一致
- Uber sensor grid 与 Q1 Tesla FSD 数据共享传闻形成行业模式

### 连锁效应
当 prompt-as-product 形成市场（💰），会反向推高对"私人工作流数据"（⚡ Uber 模型）的需求，进而触发平台方收紧"用户产出版权"政策（📈 Roblox 类合规），最终诱使工具方在用户产出物里埋暗水印争夺归属（💀 VS Code Copilot）。这是同一个循环的四个相位，**今天恰好都跳出来了**。
