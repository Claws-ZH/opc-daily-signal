→ 今天 HN Show HN：GoModel（item 47849097，168 分），用纯 Go 写的 AI 网关，明确针对 LiteLLM 的 Python 供应链问题。同一周 Trend Micro 发布 Vercel OAuth 攻击复盘（HN 341 分），证实攻击通过平台环境变量横向扩散。@crawdog 在 GoModel 评论区直接说："编译时供应链可控，是 Go 做网关的关键优势——LiteLLM 这类 Python 工具运行时供应链攻击影响更大。"

🔑 可复制动作：如果你在卖任何"AI 中间层 / 代理 / 网关 / 提示路由"产品，本周给落地页加一行 "single Go binary, signed SBOM, zero Python supply chain"，把"安全"从开发者偏好升级为转化文案。如果你在做开源工具，把 Python 版重写为 Go 单二进制 + 一份签名的 SBOM。

🔮 二阶效应：未来 90 天 SaaS 安全审查里 "Python 依赖树长度" 会成为标准问题；6 个月后企业采购清单上 "single static binary" 会变成默认筛选项；这意味着 Go/Rust 写的 AI 中间件在 B 端定价能比 Python 版高 30–50%（合规理由）。窗口期约 2 个季度，等大厂补足合规叙事后红利消退。

⚠️ 什么时候这条是错的：如果接下来 60 天没有第二次类似 Vercel 规模的 Python 供应链攻击事件曝光，"供应链恐慌"会在新闻周期外正常衰减；判断信号是 60 天后再搜 "npm supply chain attack" / "pip install trojan" 的 HN 周报点赞数有没有保持月均 >200。

## 💀 死亡信号 — 一年时间憋大产品零用户，Clickmodus 复盘

→ Reddit r/SaaS @Sad_Molasses_2146 复盘：花一年时间做"轻量级 ZoomInfo"，零用户、零收入；推倒重来后围绕自己作为小企业主的真实痛点（流量打标签）重建，1 个月做到 $7K MRR。同一天另一帖 SalesRobot 复盘 $1.2M 全周期收入，关键转折是"产品稳定性修好之前所有营销都白做"。两条都来自非创始人故事的真实操作记录。

🔑 你该检查的一件事：现在打开你正在做的项目，问"这个痛点是我自己作为用户每周都要遇到、且每次都让我火大的吗"。如果不是，限定 4 周内必须找到 5 个付费用户，否则推倒重做。同时检查：你的核心功能在最近 30 天里崩溃过几次？>2 次的话，停所有营销，先修稳定性。

🔮 二阶效应：1 个月内你能省下"为了自我说服而堆功能"的时间（典型 OPC 浪费 6 周/季度）；3 个月内你的产品故事会从"我帮你解决 X"自然变成"我也被 X 折磨过，所以做了这个"——后者转化率经验值高 2-3 倍。如果继续走伪需求路线，6 个月后大概率沉没成本陷阱锁死，无法转身。

⚠️ 什么时候可以忽略：你正在做的是企业销售，签约决策者跟使用者不是同一人——这种情况下"自己用的痛"反而误导你。判断方法是看你最近 3 笔成交里，签合同的人和实际使用产品的人是不是同一个；如果不是，本条规则不适用。

🔗 https://www.reddit.com/r/SaaS/comments/1sr217z/spent_a_year_building_a_saas_nobody_cared_about/

### 底层逻辑
OPC 最贵的成本不是金钱，是"我相信会有人需要"这种自我说服消耗的时间。Clickmodus 浪费的一年，等价于 ~2,000 小时单人成本（按 $50/小时算 = $100K 机会成本）。SalesRobot 的"先修稳定再营销"则反向提醒：投入分配错了，再多动作也归零。

### 反向检验
有人会反驳"自己挠痒不能 scale"——但 OPC 阶段不需要 scale，需要的是 5 个肯月付 $50 以上的人。这个数量级，自己挠痒完全够用。
