## 💰 单价 — Skills 作为可独立销售的微产品（addyosmani/agent-skills 一周 11,791 stars）

→ 事实：addyosmani 把「production-grade engineering skills for AI coding agents」打包成仓库，一周 1.18 万 stars，进入 GitHub Weekly Trending 前列。同期 ruvnet/ruflo（Claude agent 编排，7088）、rohitg00/agentmemory（3096）共同验证：当 Claude/Cursor/Codex 都开放 skill/extension 协议后，「资深工程师的 SOP 模板」本身就成了可被发现、被 fork、被付费订阅的产品形态。

🔑 动作：把你已经在用的 3-5 个高频工作流（部署脚本、code review checklist、调试 SOP、特定栈的 scaffolding）今天就抽成一个 skills/ 仓库，每个 skill 一个 markdown，README 写明触发场景。先免费跑流量，30 天后跑通的 top 1 个挂 Gumroad ¥99 / Lemon Squeezy $19。

🔮 二阶：3-6 个月内 skill 市场会出现「分发平台 + 评分」（类似 npm + Product Hunt 的混合体）。先发者占据「该领域权威 skill」的 SEO + 引用位，后来者要花 5-10 倍流量成本才能挤进。

⚠️ 反向检验：如果三周内仓库 stars < 50 且没有任何 issue/PR，说明你抽的不是「别人也痛」的工作流，而是个人偏好——立刻砍掉，换下一个 SOP，不要继续打磨文档。

🔗 https://github.com/addyosmani/agent-skills

### 深度补充
- **底层逻辑**：LLM agent 的能力下限被模型厂商抹平后，真正的 alpha 在「上下文工程」——即什么场景该加载什么知识。这是 OPC 多年积累的工作流首次能被结构化、被检索、被定价的窗口。
- **为什么 OPC 要关注**：传统咨询/培训内容必须靠人讲，单价高但数量天花板低；skill 化之后可复制 → 数量轴打开，单价虽降但 24h 在线分发，OPC 公式两个变量同时改善。
- **数据交叉验证**：addyosmani（11791）+ ruflo（7088）+ agentmemory（3096）三个独立项目同周冲榜，不是单点偶然。
- **连锁效应**：Cursor/Cline 等编辑器接下来会推 skill marketplace，先做仓库的人有「移植到任意平台」的优势位。

## ⚡ 时间/成本 — 26M 参数模型蒸馏出 Gemini Tool Calling 能力（Needle）

→ 事实：cactus-compute/needle 把 Gemini 的 tool calling 能力蒸馏到 26M 参数模型，HN 253 分 / 93 评。意味着 agent 框架里「决定调哪个工具 / 怎么填参数」这一步，可以彻底脱离云端 API，跑在 CPU 甚至浏览器 wasm 上。

🔑 动作：审计你正在做的任意 AI 产品里「LLM 调用的成本结构」。把「路由 / 工具选择 / 参数构造」这类不需要长上下文的步骤，本周就用 Needle 或同尺寸蒸馏模型替代。即使只替换 30% 的调用，月 API 成本可立刻降 40-60%。

🔮 二阶：6-12 个月内 agent 产品的成本竞争会从「token 单价」转向「架构分层」——谁能把链路上的轻型决策放到边缘端，谁能给客户报出竞品 1/3 的价。靠纯调用 GPT-4/Claude 的 wrapper 类产品会被双向挤压：上端被原厂吃掉，下端被边缘部署吃掉。

⚠️ 反向检验：如果替换后 task success rate 跌幅 > 8%，说明你的工具调用其实需要长上下文推理（不只是模式匹配）——回滚到原模型，把这部分作为护城河保留，而不是硬蒸馏。

🔗 https://github.com/cactus-compute/needle

### 深度补充
- **底层逻辑**：tool calling 本质是「窄域分类 + 槽填充」，远小于通用对话所需的能力。能力压缩 → 边缘部署 → 单位调用成本逼近零。
- **为什么 OPC 要关注**：API bill 是 OPC 第二大成本（仅次于自己时间）。任何能让 marginal cost 趋零的技术都直接放大「单价 × 数量 / 时间」的分子。

## 📈 数量 — 厂商敌意创造 fork 市场（OrcaSlicer-bambulab，48h 内成型）

→ 事实：Bambu Lab 关闭/限制其原生网络协议后，社区在 48 小时内组织 FULU-Foundation 把 OrcaSlicer 改造成完整支持 BambuNetwork 的 fork（HN 95 分），同帖 jeffgeerling 长文「Bambu Lab is abusing the open source social contract」冲到 HN #1（1063 分）。这是一次完整的「厂商作恶 → 用户愤怒 → 替代品 48h 上线」闭环。

🔑 动作：列出你所在垂类里 3 个「锁定型」厂商（强制订阅、关闭 API、突然涨价）。今天起在 RSS 里订阅它们的官方 changelog + 用户社区抱怨帖。一旦再次发生敌意动作，48h 内做出 fork/替代品 landing page，先抢「关键词 + 邮件列表」，产品后补。

🔮 二阶：1-3 个月这种社区怒气会沉淀为可付费迁移服务（数据导出、配置转换、托管替代）。OPC 的位置不是「写一个 fork」（社区会做免费版），而是「为不愿折腾的存量用户提供 $19-49/月 的迁移 + 托管服务」。fork 是流量入口，OPC 卖的是省时间。

⚠️ 反向检验：如果你跟踪的厂商敌意事件三个月内没有任何「替代品 GitHub stars > 500」的迹象，说明这个垂类要么用户量太小要么迁移成本太高——不要硬上，换池子。

🔗 https://github.com/FULU-Foundation/OrcaSlicer-bambulab

### 深度补充
- **底层逻辑**：Cory Doctorow 的 enshittification 周期正在加速——每个平台从「对用户好」到「对用户敌对」的窗口从 5 年压缩到 1-2 年。OPC 的机会窗口随之高频化。
- **连锁效应**：Bambu 事件之后，3D 打印社区会更警惕任何闭源固件，下一波 Prusa-style 开放硬件会重新获得溢价空间。

## 💀 成本 — Instructure 向勒索者付款（Canvas 数据被赎）

→ 事实：上市教育 SaaS Instructure 旗下 Canvas LMS 被勒索，公司选择付款，HN 223 分 / 210 评。一家年收入 5 亿美元、有专职安全团队的公司都被迫付赎金，意味着小型 SaaS OPC 的「我们数据有备份」式自信完全站不住脚——攻击者要的不是恢复，是「不公开你客户名单」。

🔑 动作：今天做一件事——把客户 PII（邮箱、付款信息、聊天记录）从主数据库里物理隔离到独立加密存储，主应用只持有 hash/token。客户列表用单独 Postgres 实例 + 至少两地异地备份 + 备份本身加密。这一动作不需要新工具，只需要 4 小时迁移。

🔮 二阶：12 个月内中小 SaaS 会被保险公司倒逼买 cyber insurance，年费 $3k-15k 起步。提前做好数据隔离 + SOC2-lite 流程的 OPC，保费可能差 3-5 倍。这是确定性的固定成本差。

⚠️ 反向检验：如果你现在的产品根本不存任何客户 PII（纯本地工具、付款走 Stripe checkout 不落库），那这条不适用——但反过来要警惕「功能升级时悄悄开始存数据」的滑坡。

🔗 https://www.insidehighered.com/news/tech-innovation/administrative-tech/2026/05/11/instructure-pays-ransom-canvas-hackers

### 深度补充
- **底层逻辑**：勒索经济学已从「加密文件」升级到「公开数据」——备份策略只防前者，无法防后者。唯一防御是先验地不持有敏感数据。
- **为什么 OPC 要关注**：单人公司没有公关团队、没有法务、没有保险，一次数据公开事件等于公司死亡。降低期望损失的唯一杠杆是降低「持有的敏感数据量」。
